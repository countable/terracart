

TerraCart

A gps driven farming rpg. Inspiration from Pokemon Go and Core keeper/Stardew valley

Core game loop.
1. Player should be rewarded for exploring a large area and finding objects which often correspond to real world map POIs.
2. Economic progression via farming, trading, hunting, exploring.
3. Safety - only encourage the player to go places that are actually safe to walk (avoid the need to walk on roads, water, private property)

Game objects:

# cells
The game is made of a grid of cells generated from vector tile data.
 - Some cells are tillable to replace with tilled soil.
 - grassland biome cells till in half the time.

# interactables
Each cell may contain one interactable object (excluding fauna). ie, rocks, trees, crops.
 - generally bare hands can be used for any interactable, but takes 9 seconds to process (modifiers on some specific interactables, ie pest HP)
 - tools: wood: 3s, copper: 2.5s, iron: 2s, gold: 1.3s, platinum: .8s, crimson: .5s, frost: .3s
 - show the tool in use when interacting, if applicable.
 - tools: net: catch animals, fishing rod: fishing, pick: mining rocks, axe: chopping trees, hoe: tilling soil, weapons: fighting pests.

# fauna
Entities that can typically move around over cells, like pets, enemies, wild animals
 - crows: move in pauses and straight lines.

# biomes
Different map polygons rasterize to tiles according to rules 

## Residential
 - Avoid interactables or fauna on private property. in residential areas, all interactables must spawn within 2 spaces of a road or path, or an area that is designated as public.
 - no animals may move onto a building footing, or road.

================================================================================
CORE
================================================================================
 - fullscreen mobile web first (desktop too), 390x844 viewport centered in the browser
 - vector map tiles from OpenFreeMap z14 (OpenMapTiles schema, no API key),
   decoded by hand (mvt.js) and cached in IndexedDB
 - Phaser 3.87 renders the world to a canvas
 - camera shows an 11x11 grid of cells, 5m each (~55m viewport). The player is
   fixed at screen centre; the world pans under them.
 - graceful network loss: a 'reconnecting' banner shows; cached tiles still
   serve from IndexedDB, and a service worker (sw.js) precaches the app shell.
 - all progress lives in one `save` object persisted to localStorage
   (debounced writes via save.js). Tiles cache separately in IndexedDB.

================================================================================
MOVEMENT & REACH
================================================================================
 - GPS (geolocation) drives the player on mobile. WASD / arrow keys move on
   desktop / localhost. The first manual key press latches a manual override
   for the session (GPS stops fighting the keyboard).
 - device compass (deviceorientation) drives a facing-arrow indicator above the
   player's head; with no compass it falls back to the last move direction.
 - REACH: you can only interact with cells within ~3 cells (cardinally) of the
   cell you stand in — a ~16m reach measured from the player's FEET, not the
   sprite centre, so the reachable area is symmetric around the visible
   character. A white silhouette outlines the reachable cells; out-of-reach
   cells are dimmed. The tap-accept test and the drawn outline use the same
   integer cell math (coords.js cellInReach) so "if it's lit, you can tap it".
 - WORK WHEEL: timed actions (chop / mine / fish / pick a worked crop / defeat a
   pest / catch an animal) show a circular progress wheel at the spot. Every
   tool type works BARE-HANDED (tier 0) at 3× the wooden-tool time — only the
   Bug Net has no bare-hands tier. You wait it out; tapping anywhere
   cancels it (and refunds any pre-charged energy). The first 150ms is
   swallowed so the launching tap can't immediately cancel.

================================================================================
ENERGY & FOOD
================================================================================
 - energy is a pool spent on hard actions and shown as ⚡cur/max (green >50%,
   yellow >25%, red below). Starting/base max is 100.
 - ENERGY_COST (items.js): till 2, plant 1, harvest 1, catch 5, rockBreak 10
   (discounted by pickaxe tier), rockPlace 1; picking wildplants/flora is free.
   Mining ore rocks costs 10 + (tier-1)*4.
 - ARMOR raises max energy: helmet +10/tier, chest +25/tier, legs +15/tier,
   boots +8/tier (per equipped tier). Equipping better armor bumps current
   energy by the delta too.
 - EAT: a persistent Eat button below the inventory eats the selected food for
   FOOD_ENERGY[id] (longgrass 2 … sunflower 150, cow 120, meat 45, etc.).
 - special food side effects: eating a rainberry waters every crop within 20m;
   eating a pairy points you to the nearest undiscovered chest for 5 minutes.
 - energy regenerates over real time: it pro-rates while the tab is closed
   (offline rest), and refills faster while you're "indoors"/at home.

================================================================================
WORLD GENERATION
================================================================================
 - each MVT tile is rasterized ONCE into a grid of 5m cells (one terrain class
   per cell), then decorated with objects, debris, creatures, and treasure.
   Rasterization is deterministic per tile (seeded) so the same place always
   generates the same world.
 - terrain cell classes group into three base biomes; each subtype gets its own
   colour so the world reads varied but cohesive:
     ROCKY/URBAN: residential, commercial (retail/hospital), industrial, road,
                  path, building tiers, pier
     FOREST:      forest, orchard, wetland
     GRASSLAND:   grass, park, school grounds, sports pitch, playground, golf,
                  farmland (allotments lumped in)
   plus water and sand. Roads split into ROAD / ROAD_MD (avenue) / ROAD_LG
   (highway); buildings into BUILDING (house) / BUILDING_MED (fort) /
   BUILDING_LARGE (castle/civic).
 - building tier from polygon area + render_height: small house (<350m² & <10m),
   fort (350+m² or 10+m), castle (1500+m² or 15+m). Each tier still places one
   house/tower sprite at the polygon centroid.
 - every tile enforces a minimum mix so trade/landmarks always exist: roughly
   ≥2% castles, ≥8% forts, ≥20% small houses (re-tiered by area rank if needed).
 - flat colour is drawn first, then a terrain sprite from the tileset is
   overlaid where mapped (grass, forest, sand, farmland, park), with auto-tile
   lookup so edges/corners pick the right frame.
 - building polygons → house/tower sprite; forest landcover → scattered trees;
   orchard → fruit trees; useful POIs → chests; fountains → wells; plus rocks,
   debris, treasure, and creatures (all below).
 - a satextract sidecar (data/) injects extra real POIs (individual trees,
   fountains, parking, street furniture) on top of the MVT layer.

================================================================================
INTERACTION MODEL
================================================================================
 - EVERYTHING is tap-driven. Walking never changes a cell. A tap does EXACTLY
   ONE thing: handlers are tried in priority order (interact.js TAP_HANDLERS)
   and the first that matches consumes the tap and stops. Tapping the player is
   treated as a tap on the player's own cell.
 - priority order (all gated to within reach of the player's cell):
     0)  cancel an in-progress work wheel
     0)  use consumable on self (tap own feet with flute / book selected)
     1)  claim a treasure X within ~7.5m
     2)  creature within 4m (defeat / catch / tame / feed / pet — see CREATURES)
     3)  pick the nearest wild plant / flower within 4m
     4)  pick up a scattered coin within ~3m
     5)  world object within reach: chest, well, tree, fruit tree, mineral rock,
         ground stack, house/tower (shop), shrine
     6)  resolve the tapped cell (bail if unloaded / out of reach)
     7)  path-stone claim (non-consuming — falls through)
     8)  building-zone tap → snap to the nearest house/tower sprite within
         30m and route to its shop (covers building footprints too large
         for the sprite's tap zone alone)
     9)  release a selected animal onto the cell
    10)  pick up a placed rock / placed scarecrow
    11)  place a held rock / scarecrow on an empty tilled-able cell
    12)  break a natural rock cell (needs energy; pickaxe speeds it up)
    13)  planted cell → harvest / advance / water / nag
    14)  refill watering can from a WATER cell
    15)  fish a WATER cell (bare-handed OK; rod speeds it) — also tops up the can
    16)  tilled empty cell + seed → plant; tilled empty cell + no seed → un-till
    17)  untilled tillable cell → till it
    18)  non-tillable terrain → flavour label (water / building / road / path)
 - the auto-till fallthrough refuses to till any cell occupied by a wildplant,
   crop, placed rock, building, tree, or chest, so a near-miss never silently
   tills under something.
 - flash text fades up over ~0.9s for routine actions; bigger "loot pops"
   (pop-in, hold, slow fade) are used for chests, treasure-X, harvests,
   pickups, and catches. Loot pops are tier-coloured.

================================================================================
FARMING LOOP
================================================================================
 - all farming is tap-driven; tools are implicit (no need to equip a hoe to
   till, though a Hoe relic discounts it).
     1) tap untilled tillable cell → TILL (brown tint). Costs 2 energy (Hoe
        relic: each tier shaves cost and adds a per-tier chance of free tilling).
     2) tap tilled empty cell with a seed selected → PLANT (consumes 1 seed).
     3) tap planted dry cell → WATER (darker tint). Stage advances 15 min later.
     4) tap watered cell after 15 min → ADVANCE one stage (becomes dry again).
     5) tap mature crop → HARVEST → +1-3 produce, plant destroyed, ~25% bonus
        seed. Growth auto-advances after 15 min if watered, even while away.
        (STAGE_HOLD_MS in app.js — intended as a tunable config constant.)
 - crop grows over 5 visual stages (cols 0..4); 4 waterings to mature.
 - any tillable cell (non-water, non-building, unoccupied) can be tilled.
 - WATERING CAN (relic): when equipped, each watering "improves" the crop. Tier
   T adds T quality tiers; refilling at water/a well/fishing banks 50 charges
   that add +2 more tiers each. At harvest, quality raises the bonus-seed chance
   (+10%/tier over base 25%) and adds +floor(qual/3) to the produce yield.
 - per-cell state: save.tilled (cell keys), save.planted (x,y,crop,stage,
   watered_t,canBoost). Tilled cells render a brown tint, darker once watered.

================================================================================
FORAGING / WILD PICKUPS
================================================================================
 - DEBRIS: every residential / park / forest / grassland / beach polygon gets a
   stable seeded density (~5-30%) and is sprinkled with one crop type:
     residential → mushroom (yards)      park → shrub       forest → shrub
     forest      → occasional nut + mushroom clusters (rarer, own RNG)
     beach/sand  → shell                 grassland → longgrass (Fern)
 - tap a wild plant / debris within 4m (and within reach) to pick it (+1).
 - some wild crops need WORK (a progress wheel): rockfruit (stone) is faster
   with a Pickaxe, shrub (woody) with an Axe — 9s bare-handed, faster by tier.
   A shrub yields WOOD, not a "shrub" item. Other wildplants are instant.
 - SURPRISE TREASURE: picking certain crops has a small chance of a bonus drop
   (e.g. rockfruit → gemfruit). Flowers pulled from flora polygons give 1
   'flowers'.
 - picked state persists in save.picked (stable per-cell id). Wild plant and
   fauna respawning is not yet implemented (future work).

================================================================================
MINING
================================================================================
 - all mining is via MINERAL ROCKS / ORE BOULDERS — objects spawned in clusters
   on quarry / industrial / residential / rocky (scree/cliff) polygons.
   Each has a requiredTier (min pickaxe tier to crack it) and either a
   yieldTier (ore) or is a plain "cave" rock.
     - cave rock: quick + cheap, drops 1-3 rockfruit + a chance of coal, with
       a small per-tier chance of a lucky bar.
     - ore rock: drops coal + exactly one bar of its tier (copper/iron/gold)
       and, on T4+, a tier-rolled gem (sapphire/ruby/emerald). Work + energy
       scale with the tier.
 - density by zone: residential = sparse low-tier clusters; industrial = dense
   mid-tier; rock landcover (scree/cliff) = dense, reaches top tiers but
   weighted toward low.
 - broken rocks persist (save.brokenRocks) so they don't regrow.
 - YOU CAN ALSO PLACE rockfruit as a stone "fence" on an empty tilled-able cell
   (costs 1 energy) and pick it back up later.

================================================================================
CHOPPING
================================================================================
 - TREES (forest landcover + injected individual trees) chop via a work wheel
   (9s bare-handed, 3s with a wood axe, faster by tier) and drop 2-3 wood.
   Chopped state persists (save.chopped); stumps
   don't block tilling.
 - SHRUBS are wildplants (see FORAGING) — an Axe speeds the chop; they drop wood.

================================================================================
ORCHARD FRUIT TREES
================================================================================
 - orchard landcover spawns fruit trees that passively produce fruit on a timer
   (~30 min per tree). When ripe, tapping the tree picks the fruit (+1); the
   tree then resets its timer. No chopping required — fruit trees are never
   destroyed. Each tree's fruit type is seeded from its cell position (apple,
   cherry, peach, banana, orange, mango, coconut, apricot).

================================================================================
FISHING
================================================================================
 - tap a WATER cell → a cast work wheel (9s bare-handed, 3s with a wood rod,
   faster by tier), then a weighted catch. Higher rod tiers improve the catch
   and rare odds. A rod also tops the watering can to full on every cast (so a
   rod owner never loses charges).
 - per cast: a "skunk" (nothing) chance that falls with rod tier (T1 ~50% →
   T7 20%); else a weighted catch — mostly fish (minnow → bass → trout → salmon
   → goldenfish, rarer more likely at high tier), with a small relic chance
   (~2%) that follows the same milestone gating as chests. Higher relic tiers
   are possible but exponentially rarer; rod tier raises the odds of better
   fish, not relics. A relic that is an upgrade auto-equips; a non-upgrade is
   discarded. 6% per cast: an old boot (junk gag).
 - fishing costs 5 energy.

================================================================================
CREATURES
================================================================================
 - spawned per tile, seeded, biased to fitting biomes; only simulated near the
   player. Kinds: chicken, cow, cat, dog (domestic); rabbit, deer, crow,
   butterfly (wilderness); slime (pest). Chickens/cows wander ~1 cell / 5s near
   their home.
 - CATCHING vs TAMING are now SEPARATE (chicken/cow/cat/dog/rabbit/butterfly):
     * CATCH = a work queue. Tap with an EMPTY hand (or any non-food item). The
       animal FLEES the player at 2 m/s while the wheel runs; if it escapes the
       viewport the catch FAILS. Finishing it puts the LIVE animal in your
       inventory. A Bug Net shortens the wheel by tier; bare hands take the
       tier-0 (9s) time. Butterflies are the one exception with no bare-hands
       tier — they REQUIRE the net.
     * TAME = feed the FAVOURITE food. Befriends the animal IN PLACE: it converts
       to a tame 'released_' pet (stays in the world, pettable / produces /
       cats follow) but does NOT enter your inventory. Favourites: chicken = any
       seed, cow = pairy, cat = milk OR any fish, dog = meat.
     * PLANT PRODUCE on chicken/cow → FEED for produce: consume the plant, get
       1 egg (chicken) / 1 milk (cow). 1-hour per-animal cooldown. A recently
       PETTED animal has a 50% chance to double the yield.
     * any other food → YUCK (wasted bite, no result).
 - PESTS — crow / deer / slime = DEFEAT via a work queue (NOT caught). Tapping
   starts the wheel; a weapon relic (sword / bow / staff) shortens it by tier,
   bare hands take the tier-0 9s. Finishing it removes the creature and drops
   its product (crow → feather, deer → meat; slimes drop nothing). Pests do NOT
   flee during the wheel.
 - RELEASE: select a caught animal and tap a passable cell to place it (chickens
   release as a flock of 4; others one at a time). Released animals persist in
   save.released and re-spawn at their spot.
 - PET: tap a released ("tame") animal with anything (or nothing) for a happy
   sound + a 10-minute petting boost (the +50% double-yield above). Petting a
   cat also makes it follow you for 5 minutes.
 - SCARECROW: a placeable item — wild crows and deer steer around it (4-cell
   aversion). Pick it back up any time.
 - CROWS additionally try to destroy nearby crops over time — scarecrows and
   defeating them are the counters.
 - SLIMES are an energy-leeching PEST (50/tile, every natural biome). They ooze
   in slow hopping drifts and are lazily drawn toward the player (~half their
   hops amble your way). A slime within ~2.5m drains 1 energy every 3s, surfaced
   as one throttled "🟢 slime drained N⚡" flash. Not catchable or feedable, but
   DEFEATABLE: tap to start a work queue (weapon shortens it; bare hands 9s),
   or just walk away to shake a swarm.

================================================================================
CHESTS (POI-driven)
================================================================================
 - useful real-world POIs (shops, food, civic, healthcare, parks, sports,
   culture, street furniture, …) each spawn a generic chest sprite at the POI.
   Chests snap to the cell grid and dedupe by name within ~80m (per tile) /
   ~120m (cross-tile). Density is data-dependent (~tens per tile in town).
 - tap a chest within reach to open it for loot driven by the POI class.
   Tier yields T1=10 / T2=5 / T3=2 per drop (low-tier street furniture is
   reduced to 3/2/1). Category weightings (T1/T2/T3), abbreviated:
     food → produce 60/30/10   commerce → seeds 70/25/5   farm → seeds 40/40/20 (+1)
     flora → flower seeds 10/30/60   civic → 30/40/30   health → 50/30/20
     park → 40/40/20   lowtier (bus/fuel/lodging/gate) → 90/8/2, reduced yields
 - chest TIER shown as a coloured diamond above the sprite: T1 bronze (lowtier),
   T2 silver (commerce/park), T3 gold (food/health/civic/farm), T4 cyan (flora).
 - 10% of opens roll a RELIC or ARMOR instead of normal loot (biased by chest
   tier, gated by your harvest/catch milestones). An upgrade auto-equips; a
   non-upgrade is discarded. Both show a reward modal.
 - BAG-FULL: if loot won't fully fit you choose "Take what fits" (chest emptied)
   or "Leave for later" (chest kept; it remembers its exact roll in
   save.chestHold so you can't re-roll for something better).
 - opened chests stay open + dimmed and persist (save.opened).
 - POI names are RUSTICIFIED for display (Hospital→Apothecary, School→Hedge
   School, Coffee→Roastery, …) and shown as a label above the chest + in the
   loot flash.

================================================================================
COIN-BURST POIs (ATM / bicycle parking)
================================================================================
 - ATM and bicycle_parking POIs render as tinted chests but DON'T give chest
   loot. Tapping scatters 8-12 individual coin pickups (+$1 each) on walkable
   cells within ~25m; each coin expires after 60s.
 - one burst per POI per UTC day (save.coinBurstClaimed); coins are in-memory
   only (not persisted). A second tap the same day flashes "already used today".

================================================================================
TREASURE MARKS (X on the ground)
================================================================================
 - claim by tapping within ~7.5m (and within reach). Reward via the unified
   rarity picker — mostly a small common (≈$1 or a common seed), sometimes an
   uncommon, rarely a rare. Found state persists in save.foundTreasures.
 - sources: the STARTER TILE seeds a trail of ~7 fixed "starter crates"
   (potato seed + wood + rockfruit) near spawn and clears a tutorial pocket of
   obstacles; every PARKING lot spawns one guaranteed X; non-starter tiles add a
   few random X marks plus extra X's beside named paths. Residential X's only
   spawn near a road.

================================================================================
PATH STONES
================================================================================
 - stepping on (or tapping) a named pedestrian-path cell claims that stone
   automatically — no item, no UI. Every 10 consecutive claimed stones on the
   same named path awards 1 coin (a 30-stone path pays 3 coins total; fewer
   than 10 pays nothing).

================================================================================
WELLS / FOUNTAINS
================================================================================
 - OSM fountains become a 'well' object: a water source on dry land. Tapping it
   refills the watering can to 50 charges (same as tapping a water tile). With
   no can yet, it just flashes flavour text.

================================================================================
ECONOMY (buying & selling)
================================================================================
 - currency = coins (save.money). Start with $25. Earn by selling, picking up
   coins, treasure/chests, and coin bursts.
 - tap a house/tower to trade. SMALL HOUSES specialise by their address digit
   (shops.js): ending 9 = BLACKSMITH (forges gear from bars/gems; +100% sell on
   gems), 2/6 = MARKET (cash-only produce stock; +50% sell on produce), 1/8 =
   TRADER (barter-only; +25% sell on anything). Other houses are PLAIN: they run
   "delivery" deals, buying 2-3 specific produce they want.
 - FORTS (BUILDING_MED): up to 5 deals/hour. CASTLES/TOWERS (BUILDING_LARGE):
   never run out, always sell RELICS (at 4× base, minus bow/staff discount), and
   are never "busy". Plain houses do 1 deal/hour. Hour buckets are per-house.
 - CASTLE TRIBUTE (locked start): every castle begins occupied by corrupt
   post-apocalyptic residents who demand a one-time TRIBUTE before they'll open
   the vault — 10 of a stable-random Tier-2 good (just 5 if it's a live animal),
   keyed per castle so it never reshuffles. Holding 10 needs a Bag relic
   (base stack cap is 9), so non-animal castles are effectively gated behind
   bags. Mirrors the wreck-house restore gate one tier up; once paid it's tracked
   in save.tributedCastles and the castle trades normally forever after.
 - OFFER MODAL: shows the item (sprite + name ×qty), a cost, and Buy/Cancel
   (greyed if you can't afford). Castles/forts may show a re-roll button
   (cost $5 × 2^rerolls).
 - BUY cost generation: 1/3 of the time a CASH price (1.2-3.0× the item value);
   2/3 of the time a BARTER deal — the shop wants 1× of a random item you hold
   whose value is high enough; if you hold nothing suitable it shows what it
   wants with the button disabled (so you learn what to bring).
 - SELL: with a stack selected, tapping a house sells one. Sale price =
   PRICES[id] × sword multiplier (0.5× with no sword → 1.0× at Frost) × the
   shop's specialty bonus. Selling gear you already own better of is half value.
 - bow/staff relics shrink the buy markup (best tier wins; T7 = buy at par).

================================================================================
GEAR: RELICS & ARMOR
================================================================================
 - 7 MATERIAL TIERS for all gear: Wood, Copper, Iron, Gold, Platinum, Crimson,
   Frost (cost + effect climb together). One relic per slot, one armor per slot;
   buying/forging an equal-or-lower tier into an occupied slot is refused.
 - RELIC SLOTS and effects (items.js RELIC_DEFS):
     pick   — lets you break rocks; tier shortens the wheel + cuts energy
     axe    — lets you chop trees; tier shortens the wheel
     hoe    — cheaper tilling, per-tier chance of free
     can    — watering improves crops (see FARMING); refill banks 50 charges
     rod    — fish from water; tier raises catch + rare odds
     bugnet — speeds the animal-catch wheel by tier; butterflies REQUIRE it
     bags   — raises the per-item stack cap (9 with none → 249 at T7)
     ring   — +5%/tier chance to bump chest loot up a tier
     amulet — projects a GHOST (see below); higher tier = faster + cheaper
     sword  — raises sell prices (0.5× → 1.0× at Frost)
     bow / staff — lower buy prices (best tier wins; T7 = par)
 - ARMOR SLOTS (helmet / chest / legs / boots) only raise MAX ENERGY per tier.
 - GHOST MODE (amulet): an on-screen pad projects a fast ghost you steer to
   SCOUT ahead while your real body stays put. Energy drains per cell moved
   (~2/cell at T1, falling to ~0.3/cell at T7). Releasing the pad collapses
   the ghost back to your body. Speed scales ~8×→24× walk with amulet tier.
 - TIER GATING: rarer relic/armor tiers only drop/sell after milestones —
   harvest a sunflower → unlock Gold, catch a cow → Platinum, harvest a
   fireflower → Crimson, harvest an iceflower → Frost. Chest tier also caps the
   relic tier it can give (bus chests cap at Wood).

================================================================================
CRAFTING: BLACKSMITH FORGE & MAGIC SHRINE
================================================================================
 - STARTER BLACKSMITH: the small house nearest your home is forced to be a
   smithy that forges all three wooden tools (Pickaxe, Axe, Hoe) in any order,
   5 wood each. The starter crate seeds the first 5 wood. Once all three are
   owned it becomes a normal relic forge.
 - BLACKSMITH FORGE (any address-9 house, forts, castles): forge gear from
   materials. Tools / weapons / armor cost max(5, tier) of the tier-matched bar
   (T1 = wood). JEWELRY (ring/staff/amulet) costs 2^(tier-2) gems + 1 bar
   (ring→ruby, staff→emerald, amulet→sapphire); no T1 jewelry.
 - SMELTING (blacksmith, T5+ only): the top bars are smelted from flowers +
   the lower bar — platinum = sunflower + gold, crimson = fireflower + platinum,
   frost = iceflower + crimson. Copper/iron/gold bars come from mining.
 - MAGIC CRAFTING SHRINE: a per-game altar near spawn. Tap it to either LEVEL
   UP or run a TRANSFORM:
     * level up (L1→L6): pay 5× of three tier-matched items
       (e.g. L1→L2 = 5 potato + 5 egg + 5 coal; L5→L6 = 5 fireflower +
       5 fireflower seed + 5 platinum bar).
     * each level unlocks one 1-produce→1-bar transform: L2 rainberry→iron,
       L3 coffee→gold, L4 sunflower→platinum, L5 fireflower→crimson,
       L6 iceflower→frost (endgame). The shrine sprite upgrades each level.

================================================================================
CONSUMABLES
================================================================================
 - FLUTE: select it and tap your own feet to play — lures wandering creatures
   within 15 cells toward you (flash reports how many respond). Consumed on use.
 - BOOK: select + tap your feet to read — ~50% a play tip, ~50% a directional
   hint to the nearest unopened chest ("about N paces northwest"). Consumed.

================================================================================
INVENTORY
================================================================================
 - an effectively infinite list shown 5 slots per page with ◀ ▶ paging.
 - tap a slot to select; selection persists. Tools/relics are NOT inventory
   items — they're equipped gear shown elsewhere.
 - per-item STACK CAP starts at 9 and rises with the Bags relic (up to 249).
 - holds seeds, produce, minerals/bars/gems, animals, fish, fruit, wild
   pickups, consumables, scarecrows. Stacks increment on pickup; selecting +
   tapping drives plant / release / place / feed / eat / sell.
 - starter inventory: a stack of potato seeds (+ the starter crate trail).
 - ITEMS ALWAYS RENDER AS THEIR GAME-ART SPRITE on every surface (map,
   inventory, shop, toast, sign). Emoji is reserved for non-item UI only
   (see docs/QC_RULES.md §1).

================================================================================
ITEMS & RARITY
================================================================================
 - every item has a baseTier 1..7 (items.js BASE_TIER) feeding a unified rarity
   picker (rarity.js) used by chests, treasure, and fishing.
 - rough catalog (see items.js for the authoritative list + prices):
     crops/produce: potato, rockfruit, rainberry, pairy, nut, berry, cress,
       onion, gemfruit, coffee, sunflower, fireflower, iceflower (+seeds)
     animal output: egg, milk        wild: longgrass(Fern), flowers, mushroom, shell
     fauna drops:   meat, rabbit_pelt, crow_feather       junk: boot
